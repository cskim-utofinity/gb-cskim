신용평가 업무 및 데이터 분석 과정에서의 효율성을 높이기 위해 개발된 사내용 Python 유틸리티 패키지입니다. 반복적인 로깅, 대용량 데이터의 메모리 관리, NW 시스템 추출 결과의 Pandas/Polars DataFrame 변환, 요약항목 스키마(TOML) 관리 기능을 제공합니다.

Python ≥3.11, <3.12 v0.2.0 Pandas + Polars 스키마 번들 내장
Overview

패키지 구성

import cskim as cs 한 줄로 4개 모듈의 모든 기능을 패키지 레벨에서 바로 호출할 수 있습니다.

utils

시간 및 로깅

KST 기준 타임스탬프 생성 및 단계별 실행 로그 출력. 다른 모듈의 기반 모듈.

현재 메모리 점유량 확인, GC 강제 실행, malloc_trim으로 OS에 메모리 즉시 반납.

data

DataFrame 도구

로깅 내장 CSV/Parquet 읽기·쓰기, concat, merge. pandas/polars 양쪽 지원.

nw

사내 NW 출력 파일(.gz) 직접 파싱. 쿼리·스코어·요약항목 세 종류, TOML 스키마 내장.

스키마 셋업 (v0.2.0+)

read_query / read_scr / read_css가 사용하는 컬럼·필드 정의(TOML)는 패키지에 번들로 포함되어 있어 설치 직후 별도 셋업 없이 동작합니다.

번들 기본값으로 바로 사용 가능 queries.toml / scores.toml / css.toml 3종이 패키지 내부(cskim/_schemas/)에 포함되어 있습니다.

사내 표준 최신본이나 신규 항목을 쓰려면 외부 파일을 두고 경로만 지정합니다.

import cskim as cs

# 외부 스키마 디렉토리 지정 (파일 단위 override)
cs.set_schema_dir("/path/to/schemas")

# 파일 수정 후 캐시 갱신
cs.reload_schemas()

# 번들 스키마를 외부 경로로 복사 (최초 구성용)
cs.export_schemas("~/my_schemas")

Polars 백엔드

모든 read_*, concat, merge 함수에 backend='polars'를 전달하면 pl.DataFrame을 반환합니다.

backend 값출력 타입비고
'pandas'pd.DataFrame기본값. 기존 코드와 호환
'polars'pl.DataFramePolars 네이티브. 더 빠른 처리
concat · merge 주의 backend='polars'로 호출할 때는 입력 리스트의 모든 원소도 pl.DataFrame이어야 합니다. 
import polars as pl

# pd.DataFrame → pl.DataFrame
pl_df = pl.from_pandas(pd_df)

# pl.DataFrame → pd.DataFrame
pd_df = pl_df.to_pandas()

tsnow()

현재 시각을 YYYY-MM-DD HH:MM:SS 형식의 문자열로 반환합니다.

cs.tsnow() → str

print(now)  # 2026-06-08 14:00:00

cskim.system

chk_memory()

현재 파이썬 프로세스의 실제 메모리(RSS) 점유량을 GB 단위로 출력합니다.

cs.chk_memory(result=False, memory_clean=True) → int | None
파라미터타입기본값설명
resultboolFalseTrue 시 사용량(Bytes) 반환
memory_cleanboolTrue측정 전 GC + malloc_trim 수행
 
# 단순 출력

# [2026-06-08 14:00:00] 현재 메모리 사용량은 1.25GB 입니다.

# 반환값으로 조건 분기

memory_clean()

파이썬 GC를 즉시 강제 실행합니다. del df 후 메모리가 즉시 해제되지 않을 때 사용합니다.

cs.memory_clean() → None

release_memory()

C 라이브러리의 malloc_trim을 호출하여 파이썬이 점유 중인 빈 메모리 영역을 OS로 즉시 반환합니다.

cs.release_memory() → None
  • Linux 최적화: JupyterLab 서버(Linux) 환경에서 가장 효과적으로 작동
  • Windows 안전: Windows에서는 조용히 skip합니다(에러·메시지 출력 없음). 반복 호출해도 불필요한 출력이 없습니다.
# 대용량 병합 완료 후 시스템에 메모리 반납
cs.release_memory()

for file in file_list:

cskim.data

Pandas/Polars의 핵심 기능을 확장하여 자동 로깅, 메모리 모니터링, 자동 dtype 지정 기능을 제공합니다. 모든 함수에 backend 파라미터가 있으며 기본값('pandas')은 기존 동작과 동일합니다.

concat()

여러 DataFrame을 하나로 합칩니다. 결합 전후 행/열 상태를 로그로 출력하며 ignore_index=True가 기본값입니다.

merge() merge_seq()

초기 DataFrame에 여러 DataFrame을 순차적으로 Left Join합니다. 병합 중 오류가 발생하면 안내 메시지 출력 후 예외를 전파합니다(조용한 실패 없음).

이름 변경 안내 기존 merge_seq는 merge로 이름이 바뀌었습니다. merge_seq도 동일하게 동작하지만, 호출 시 안내 메시지가 출력됩니다. 앞으로는 merge를 사용해 주세요.
cs.merge(merge_dfs, on_key=['rbc_no','base_date'], how='left', backend='pandas')
파라미터타입기본값설명
merge_dfslist—병합할 DataFrame 리스트. 첫 번째가 기준
on_keystr/list['rbc_no','base_date']조인 키
howstr'left'left / outer / inner. polars에서 outer는 자동으로 full로 변환
backendstr'pandas''pandas' 또는 'polars'
 
# pandas (기본)
final_df = cs.merge(
    merge_dfs=[base_df, score_df, css_df],
    on_key=['rbc_no', 'base_date']
)

# polars
final_pl = cs.merge(
    merge_dfs=[pl_base, pl_score, pl_css],
    on_key=['rbc_no', 'base_date'],
    backend='polars'
)

스키마 탐색 체계

각 스키마 파일은 아래 순서로 처음 발견되는 곳에서 로드됩니다. 외부 디렉토리에 일부 파일만 두면 그 파일만 외부본이 적용되고, 나머지는 번들이 사용됩니다.

1
set_schema_dir 지정 경로 cs.set_schema_dir("/path") — 세션 전역, 최우선
2
DA_SCHEMA_DIR 환경변수 export DA_SCHEMA_DIR=/path (bash) · $env:DA_SCHEMA_DIR="..." (PowerShell)
3
패키지 번들 기본값 cskim/_schemas/ — 항상 존재, 최종 fallback
# 외부 스키마 지정
cs.set_schema_dir("/path/to/schemas")

# 파일 수정 후 캐시 갱신
cs.reload_schemas()

요약항목 리스트 파일 자동 구성

NW는 결과 파일과 함께 요약항목 리스트 파일을 생성합니다. 이 파일을 읽어 수천 개 컬럼명을 직접 나열하지 않고 total_cols를 자동 구성합니다.

  • 파일 대응: 결과 N001_CSALINF_OCR2.260601.gz ↔ 리스트 N001_CSALINF_OCR2.260601_LIST.gz
  • 리스트 내용: comma 구분 한 줄 — 추출포맷, 요약항목개수, 항목1, 항목2, ...
  • 구성 결과: total_cols = key_cols + 리스트항목
  • 지원 포맷: 현재 css_00009(comma)만 지원
# 가장 간단한 사용 — 리스트 파일 자동 유도 + 기본 key_cols
df_css = cs.read_css(file_path="N001_CSALINF_OCR2.260601.gz")

# 식별자가 사업자번호인 경우 key_cols 지정
df_css = cs.read_css(
    file_path="N001_CSALINF_OCR2.260601.gz",
    key_cols=["biz_no", "base_date"]
)

# 기존 방식 (total_cols 직접 입력) — 그대로 동작
df_css = cs.read_css(total_cols=my_vars, file_path="N001_CSALINF.gz")

# NW번호 자동 탐색 + polars
df_pl = cs.read_css(nw_no="12345", backend='polars')

스키마 운영 유틸 v0.2 신규

번들 스키마를 외부로 복사하거나, 신규 요약항목을 css.toml에 일괄 반영하는 두 유틸입니다.

export_schemas()

번들 스키마 3종(queries.toml / scores.toml / css.toml)을 지정 경로로 복사합니다. 외부 스키마를 처음 구성하거나 번들 최신본으로 리셋할 때 사용합니다.

cs.export_schemas(dest_dir: str) → None
cs.export_schemas("~/my_schemas")
# → ~/my_schemas/queries.toml, scores.toml, css.toml 복사

update_css_schema()

신규 요약항목을 정보구분, 요약항목명, dtype 3열 CSV 한 장으로 css.toml에 일괄 반영합니다.

cs.update_css_schema(src: str, dest: str, *, overwrite=False) → None
파라미터타입기본값설명
srcstr—입력 CSV 경로 (정보구분, 요약항목명, dtype 3열)
deststr—수정할 css.toml 경로
overwriteboolFalse기존 항목 덮어쓰기 여부. False면 신규 항목만 추가
  • 신규 항목은 해당 정보구분 섹션 안에 코드 사전순으로 삽입. 없는 정보구분이면 파일 끝에 새 섹션 생성
  • 기존 주석·항목은 그대로 보존
  • dtype 유효성 검사 내장 — 허용 목록 외 값이 하나라도 있으면 전체 업데이트 보류 후 문제 항목 보고
cs.update_css_schema("new_items.csv", dest="~/my_schemas/css.toml")
# dtype 오류 시:
# [update_css_schema] 유효하지 않은 dtype - 업데이트 보류
#   NEW_ITEM_003: dtype 누락 → 허용값: str, int8, int16, int32, int64, float32, float64
#   NEW_ITEM_005: 'integer' → 허용값: ...
운영 워크플로 신규 요약항목 추가 → update_css_schema로 css.toml 갱신 → cs.set_schema_dir로 외부본 지정 → cs.reload_schemas()로 캐시 갱신 → 즉시 반영
