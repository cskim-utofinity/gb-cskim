설치 방법

GitLab 저장소에서 직접 설치합니다.

# 최신 버전 (main 브랜치)
uv add git+https://cbgitlab.nicecredit.co.kr/cskim/cskim_py_lib.git

# 특정 버전 (태그 지정)
uv add git+https://cbgitlab.nicecredit.co.kr/cskim/cskim_py_lib.git@v0.2.2

업그레이드하려면, 작업폴더 pyproject.toml의 cskim 의존성에 박힌 rev 값을 새 태그로 바꾼 뒤 uv sync 합니다.

cskim = { git = "https://cbgitlab.nicecredit.co.kr/cskim/cskim_py_lib.git", rev = "v0.2.2" }
uv sync
분석 안정성을 위해 각 작업폴더는 rev = "v0.2.1"처럼 버전을 고정해 설치합니다. 이 경우 uv lock --upgrade-package cskim은 rev가 고정돼 있어 동작하지 않으므로, 위처럼 rev 값을 직접 바꾸고 uv sync 해야 합니다.

스키마 셋업 (v0.2.0+)

read_query / read_scr / read_css가 사용하는 컬럼·필드 정의(TOML)는 패키지에 번들로 포함되어 있어 설치 직후 별도 셋업 없이 동작합니다.

번들 기본값으로 바로 사용 가능 queries.toml / scores.toml / css.toml 3종이 패키지 내부(cskim/_schemas/)에 포함되어 있습니다.

사내 표준 최신본이나 신규 항목을 쓰려면 외부 파일을 두고 경로만 지정합니다.

import cskim as cs

# 외부 스키마 디렉토리 지정 (파일 단위 override)
cs.set_schema_dir("/path/to/schemas")

# 파일 수정 후 캐시 갱신
cs.reload_schemas()

# 번들 스키마를 외부 경로로 복사 (최초 구성용)
cs.export_schemas("~/my_schemas")

Polars 백엔드

resolve_base_dir() v0.2.1

data의 read_csv와 nw의 read_query·read_scr·read_css가 파일을 찾을 때 쓰는 기준 경로(base_dir) 처리 규칙의 단일 진실원천입니다. 네 함수가 완전히 동일하게 동작하며, resolve_base_dir(base_dir)는 그 규칙으로 확정되는 최종 기준 폴더를 직접 돌려주는 공개 API입니다 — 노트북에서 "지금 어디를 보고 있나"를 확인할 때 사용합니다.

cs.resolve_base_dir(base_dir: str | None) → Path

두 가지 동작 — 지정 vs 자동

base_dir동작
값 지정자동 탐색을 하지 않고 그대로 사용. 상대경로면 현재 작업 폴더(CWD) 기준으로 풀림
None (기본값)아래 우선순위로 처음 "존재하는" 폴더를 자동 선택

자동 탐색 우선순위 (base_dir=None)

순위탐색 위치조건
1../data/rawCWD 폴더명이 정확히 notebooks일 때만
2../dataCWD 폴더명이 정확히 notebooks일 때만
3./data/raw항상
4./data항상
5./ (현재 폴더)항상

위에서부터 검사해 처음 존재하는 폴더를 채택합니다. 1·2순위는 CWD 폴더명이 정확히 notebooks(대소문자 무관)일 때만 후보에 추가되어, 작업폴더/notebooks에서 실행하든 작업폴더에서 바로 실행하든 동일하게 작업폴더/data/raw를 인식합니다(작업폴더 실행 시 상위의 엉뚱한 data를 잘못 읽지 않음). 5순위 ./는 항상 존재하므로 자동 탐색은 반드시 하나를 반환합니다.

파일명만 vs 경로 포함

base_dir은 파일명만 입력했을 때만 결합됩니다. 경로가 한 칸이라도 포함되면(./sub/x.gz, ../x.gz, 절대경로 등) base_dir은 무시되고 입력 경로를 그대로 씁니다. nw의 세 함수는 파일명 대신 식별자(nw_id+query_id / nw_no+score_cd / nw_no)만 줘도 base_dir 폴더 안을 훑어 파일을 찾습니다.

흔한 함정 — 노트북에서 상대경로 base_dir 작업폴더/notebooks에서 base_dir="data/raw"를 주면 작업폴더/notebooks/data/raw를 찾다가 "해당 디렉토리가 존재하지 않습니다"가 납니다. 데이터가 작업폴더/data/raw에 있다면 base_dir을 생략(자동 탐색이 이 경우를 처리)하거나 "../data/raw"·절대경로를 쓰세요. 자동 탐색은 폴더의 존재만 보고 그 안의 파일 유무는 보지 않습니다. 
import cskim as cs

# 지금 자동으로 잡히는 기준 폴더 확인
cs.resolve_base_dir(None)

# 지정값이 실제로 어디로 풀리는지 확인 (상대경로는 CWD 기준!)
cs.resolve_base_dir("../data/raw")
못 찾을 때 안내 · _LIST 파일 식별자 자동 탐색이 실패하면 base_dir을 지정하지 않은(None) 경우에 한해 위 우선순위 경로 목록을 오류 안내에 덧붙입니다. read_css의 요약항목 리스트(_LIST) 파일은 이 우선순위를 다시 타지 않고 본파일이 발견된 폴더(본파일 옆)에서 찾습니다.

release_memory()

C 라이브러리의 malloc_trim을 호출하여 파이썬이 점유 중인 빈 메모리 영역을 OS로 즉시 반환합니다.

cs.release_memory() → None
  • Linux 최적화: JupyterLab 서버(Linux) 환경에서 가장 효과적으로 작동
  • Windows 안전: Windows에서는 조용히 skip합니다(에러·메시지 출력 없음). 반복 호출해도 불필요한 출력이 없습니다.

cs.release_memory()

요약항목 리스트 파일 자동 구성

NW는 결과 파일과 함께 요약항목 리스트 파일을 생성합니다. 이 파일을 읽어 수천 개 컬럼명을 직접 나열하지 않고 total_cols를 자동 구성합니다.

  • 파일 대응: 결과 N001_CSALINF_OCR2.260601.gz ↔ 리스트 N001_CSALINF_OCR2.260601_LIST.gz
  • 리스트 내용: comma 구분 한 줄 — 추출포맷, 요약항목개수, 항목1, 항목2, ...
  • 구성 결과: total_cols = key_cols + 리스트항목
  • 지원 포맷: 현재 css_00009(comma)만 지원
  • 탐색 위치: css_list 생략 시 리스트 파일은 base_dir 우선순위를 다시 타지 않고 본파일이 발견된 폴더(본파일 옆)에서 {stem}_LIST{suffix}로만 찾습니다
# 가장 간단한 사용 — 리스트 파일 자동 유도 + 기본 key_cols
df_css = cs.read_css(file_path="N001_CSALINF_OCR2.260601.gz")

# 식별자가 사업자번호인 경우 key_cols 지정
df_css = cs.read_css(
    file_path="N001_CSALINF_OCR2.260601.gz",
    key_cols=["biz_no", "base_date"]
)

# 기존 방식 (total_cols 직접 입력) — 그대로 동작
df_css = cs.read_css(total_cols=my_vars, file_path="N001_CSALINF.gz")

# NW번호 자동 탐색 + polars
